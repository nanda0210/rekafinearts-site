========================================
TROUBLESHOOTING
========================================

If npm says package.json not found:
- you are in the wrong folder
- go to:
  /Users/nandamac/myprojects/NewBackupAPR25/rekafinearts-site

If category images do not show:
1. confirm files are in the right category folder
2. rerun:
   python3 generate-image-data.py
3. refresh frontend

If hero image does not show:
- verify file is inside:
  public/images/hero-open
- confirm exact filename in App.jsx

If backend fails to start:
- make sure you run:
  node server.cjs
- not node server.js

If DB file is missing:
- run:
  node init-db.cjs

========================================
ZIP AND MOVE TO HOSTING PORTAL
========================================

Purpose:
Package the frontend build and upload it to cPanel hosting.

Important:
- Upload only the built frontend
- Do NOT upload the full project folder
- Do NOT upload node_modules, src, public, database, or backend files
- Only upload the build output from dist

----------------------------------------
STEP 1: BUILD THE FRONTEND
----------------------------------------

cd /Users/nandamac/myprojects/NewBackupAPR25/rekafinearts-site
npm run build

This creates:
 /Users/nandamac/myprojects/NewBackupAPR25/rekafinearts-site/dist

----------------------------------------
STEP 2: ZIP THE DIST CONTENTS
----------------------------------------

Recommended:
zip the contents inside dist so they extract directly into public_html

cd /Users/nandamac/myprojects/NewBackupAPR25/rekafinearts-site/dist
zip -r ../dist-contents.zip .

This creates:
 /Users/nandamac/myprojects/NewBackupAPR25/rekafinearts-site/dist-contents.zip

----------------------------------------
STEP 3: OPEN HOSTING PORTAL / CPANEL
----------------------------------------

Log in to cPanel / hosting portal

Go to:
 File Manager

Open:
 public_html

----------------------------------------
STEP 4: UPLOAD ZIP FILE
----------------------------------------

Inside public_html:
- click Upload
- upload:
  dist-contents.zip

After upload completes:
- go back to File Manager
- stay inside public_html

----------------------------------------
STEP 5: EXTRACT ZIP
----------------------------------------

Inside public_html:
- select dist-contents.zip
- click Extract

After extraction, public_html should contain:
- index.html
- assets/

Important:
There should NOT be a nested dist folder unless intentionally desired

Correct:
 public_html/index.html
 public_html/assets/...

Wrong:
 public_html/dist/index.html
 public_html/dist/assets/...

----------------------------------------
STEP 6: CLEAN UP
----------------------------------------

Optional:
- delete dist-contents.zip after extraction

----------------------------------------
STEP 7: VERIFY WEBSITE
----------------------------------------

Open the public domain in browser

Expected:
- homepage loads
- styles load
- JS loads
- images referenced in frontend build load

----------------------------------------
IF SITE DOES NOT LOAD CORRECTLY
----------------------------------------

1. Check extracted structure
Make sure files are directly inside public_html, not inside public_html/dist

2. If extracted into dist folder by mistake
Move:
 public_html/dist/index.html  -> public_html/index.html
 public_html/dist/assets      -> public_html/assets

3. Refresh browser

----------------------------------------
STATIC DEPLOYMENT LIMITATION
----------------------------------------

Uploading dist to public_html deploys only the frontend.

These backend features will NOT work in static-only hosting unless backend is deployed separately:
- likes
- comments
- admin page actions
- SQLite database APIs
- server.cjs endpoints

Frontend-only hosting supports:
- page layout
- menus
- category display
- static assets
